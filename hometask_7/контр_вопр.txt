1. Что делает команда from module import *? Какие проблемы могут возникнуть при её использовании?
Позволяет подключить *все переменные из модуля.
*есть два варианта: 1) если в модуле определена переменная __all__(список атрибутов, которые могут быть
подключены), то будут подключены только атрибуты из этого списка; 2) иначе будут подключены все атрибуты,
не начинающиеся с нижнего подчёркивания.
Переменные, имеющие одинаковые имена, перезаписываются, поэтому импортирование всех атрибутов модуля может
нарушить пространство имён нашей программы.
2. Какое определение функции будет использовано при возникновении конфликта имен?
Последнее, в результате переопределения.
3. В чем проблемы использования декораторов? Можно ли процесс декорирования обернуть вспять?
Декораторы увеличивают время работы функции (так как декораторы - это тоже функции, в которые вложены
декорируемые функции). Декорирование нельзя обернуть вспять в общем случае.
4. Как порядок декорирования влияет на результат работы функции?
Сначала будет выполняться действие ближайшего к функции декоратора. Он будет определённым образом изменять
поведение функции и возвращать новую. Следующий декоратор уже будет работать с новой функцией и так далее. Поэтому,
конечно, порядок декораторов будет влиять на итоговый результат.
